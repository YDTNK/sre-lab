Architecture

構成

Reliability Demo APIを中心に、公開ページ、API、監視、CI/CD、Issue運用のつながりを整理します。

Current scope

Pages · Workers · Grafana · GitHub API / Worker behavior unchanged

System Overview

役割

05 components

01
PG

Cloudflare Pages

静的なポートフォリオ画面を配信します。訪問者はここからAPI、運用ドキュメント、GitHub上の証跡へ移動します。

02
API

Cloudflare Workers

Reliability Demo APIを提供します。ヘルスチェック、遅延、意図的なエラー、fallback、statusを扱います。

03
MON

Grafana Cloud

Synthetic MonitoringとAlertingを担当します。外形監視でAPIの状態を確認し、異常時の通知起点になります。

04
CI

GitHub Actions

CI、デプロイ、Smoke Testを自動化します。Pull Requestから本番反映までの変更経路を管理します。

05
ISS

GitHub Issues

アラート、障害対応、改善タスクを記録します。運用上の判断や対応履歴をIssue / PR単位で追跡します。

Request Flow

リクエストの流れ

Frontend to API

  1. 01
    USER
    Visitor

    公開サイトにアクセスします。

  2. 02
    PAGE
    Cloudflare Pages

    静的HTML/CSSを配信します。

  3. 03
    API
    Workers API

    Reliability Demo APIへリクエストします。

  4. 04
    JSON
    JSON response

    ヘルス状態やデモ応答を返します。

Monitoring / Alert Flow

監視とアラートの流れ

Synthetic check to Issue

  1. 01
    SYN
    Grafana Synthetic Monitoring

    /api/health や /api/status を外形監視します。

  2. 02
    ALRT
    Alert rule

    失敗や遅延悪化を条件として検知します。

  3. 03
    HOOK
    Worker webhook

    Grafana通知を受け取り、Issue化に必要な情報を整えます。

  4. 04
    ISS
    GitHub Issue

    アラート内容を運用タスクとして記録します。

  5. 05
    RUN
    Runbook

    確認手順、切り分け、復旧判断へつなげます。

CI/CD Flow

変更反映の流れ

PR to production

  1. 01
    PR
    Pull Request

    変更内容をレビュー可能な単位に分けます。

  2. 02
    CI
    GitHub Actions CI

    静的チェックや検証を実行します。

  3. 03
    DEP
    Worker deploy

    Cloudflare Workersへ反映します。

  4. 04
    TEST
    Smoke test

    本番APIの基本応答を確認します。

  5. 05
    PROD
    Production verification

    本番反映後の状態を記録します。

Technology Choices

技術選定

Current stack

Cloudflare Pages / Workers

小さな公開Web/APIポートフォリオを軽量に公開し、APIの応答やデモ挙動を管理しやすくするために使用しています。

Grafana Cloud

外形監視とアラート設計を見せるために使用しています。SLOや障害対応の入口として扱います。

GitHub Actions / Issues

変更管理、CI/CD、アラート記録、改善タスク管理をGitHub上に集約します。
