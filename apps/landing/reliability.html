Endpoint Behavior

エンドポイント挙動

Expected responses

01 GET /api/health checking

本番稼働確認のPrimary uptime checkです。

02 GET /api/status checking

サービスの状態、構成、アクティブなエンドポイントを確認します。

03 GET /api/fallback checking

縮退運転を想定したfallback demoです。

04 GET /api/slow?delayMs=1000 checking

制御された遅延を返すlatency demoです。

05 GET /api/error checking

意図的な500エラーデモです。通常のAvailability計算には含めません。

Operations Flow

運用対応の流れ

Signal to follow-up

  1. 01
    SYN
    Synthetic check

    /api/health や /api/status を外形監視します。

  2. 02
    ALRT
    Alert rule

    連続失敗や遅延悪化を検知します。

  3. 03
    ISS
    GitHub Issue

    アラートを運用タスクとして記録します。

  4. 04
    RUN
    Runbook

    切り分け、確認、復旧判断を実施します。

  5. 05
    PR
    Follow-up PR

    原因修正や再発防止をPull Requestで追跡します。
