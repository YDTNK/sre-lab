# Postmortem テンプレート

## 概要

```text
Title:
Date:
Service:
Severity:
Status:
```

## 影響

サービスや利用者への影響を記録します。

```text
対象:
内容:
時間:
```

## 検知

最初に気づいたきっかけを記録します。

```text
Detected by:
First signal:
Alert / issue link:
```

## タイムライン

| Time | Event |
|---|---|
| YYYY-MM-DD HH:mm | 内容を記録する |

## 原因

技術面と運用面に分けて整理します。

```text
Technical cause:
Operational cause:
```

## 対応

実施した対応を記録します。

```text
Initial action:
Service action:
Communication:
```

## 良かった点

- 検知できたこと
- 切り分けに役立ったこと
- 既存のRunbookや監視が役立ったこと

## 改善点

- 判断で迷った点
- 手順を明確にすべき点
- 追加すべき監視や自動化

## フォローアップ

| Action | Owner | Issue / PR | Status |
|---|---|---|---|
| 改善内容を記載する | TBD | TBD | open |

## SLOへの影響

```text
Affected SLO:
Error budget impact:
SLO missed: yes/no
```

## 学び

次回に向けて、検知、判断、対応を改善するための学びを記録します。
